## 소프트웨어의 정의
- 소프트웨어는 하드웨어를 동작 시켜 사용자가 작업을 편리하게 수행하게끔 하는 프로그램과 자료구조의 총칭.
- 개발 과정에서 생성되는 모든 산출물과 각 단계에서 만들어지는 문서와 사용자 매뉴얼을 모두 포함한다.

## 소프트웨어의 특징

### 제조가 아닌 개발
- 소프트웨어 개발과정은 개인능력에 따른 차이가 크다
### 소모가 아닌 품질 저하
- 하드웨어와 달리 닳지 않으며 시간이 지나도 고장 빈도가 높지않다.
- 사용 시작 단계부터 사용자의 요구가 계속 발생한다.

### 공학적으로 잘 작성된 소프트웨어란..?
- 사용자의 요구대로의 작동
- 원하는 기능을 수행 가능해야한다
- 신뢰성이 높고 효율적이어야 한다.
- 유지보수성이 확보되어야 한다.
- 문서화가 잘되고 적당한 사용자 인터페이스로 사용하기 편리해야한다.
- 남이 알아보기 쉬워야하며 경제적이어야 한다.

### 소프트웨어의 생산성(Productivity)
- 소프트웨어의 생산성 : 투입된 비용, 노력 등에 대한 생산성을 의미한다
- 생산성에 미치는 요소 : 개발자의 능력, 원할한 팀의 의사전달, 제품 복잡도
- 소프퉤어의 문서 표준.

### 시스템
- 공통의 목적이나 목표를 달성하기 위하여 여러가지 상호 관련된 요소들을 유기적으로 결합하는 것.
- 입력 / 처리 / 출력 / 제어

### 소프트웨어 공학의 학문적 정의 
>> 품질 좋은 소프트웨어를 경제적으로 개발하기 위한 계획을 세우고, 개발하고 , 유지 및 관리 하는 전 과정에서 공학, 과학 및 수학적 원리와 방법을 적용하여 필요한 이론과 기술 및 도구들에 대해 연구하는 학문


### 소프퉤어의 생명주기의 개념
- 개발 방법론의 바탕이 되는 것으로 소프트웨어를 개발하기 위해 정의하고 운영, 유지보수 등의 과정을 각 단계별로 나눈 것.
- 소프트웨어 생명주기의 역할 : 개발계획 수립/ 표준화/ 관리 가능
- 정의 / 개발 / 유지보수

### 일반적인 소프퉤어 생명주기 모형의 종류

## 주먹구구식 모델
- 공식적인 가이드라인 없이 프로세스가 없는 개발 방식
- 즉흥적 개발, 문서화된 산출물이 없어 관리 및 유지보수가 매우 어렵다.

## 선형 순차적 모델(폭포수 모델)
- 물이 떨어지듯이 다음 단게로 넘어가는 모델
- 초기에 개발된 전통적인 모델
- 표준 프로세스를 정하여 소프트웨어를 순차적인 개발을 함.

### 폭포수 모델의 개발 절차
- 계획 , 분석, 설계, 구현, 테스트, 유지보수의 각 단계가 하향식으로 진행.
- 단계가 끝날때마다 확실한 매듭을 짓고 그 결과 확인 후에 다음 단계로 넘어감
- 요구분석 명세서라는 문서를 작성
- 폭포수 모형 : 타당성 검토 -> 계획 - > 요구분석 -> 설계 -> 구현(코딩) -> 시험(검사, 테스트) -> 유지보수

### 폭포수 모델의 장점과 단점 
- 장점 : 관리 용이, 체계적인 문서화 가능, 요구사항 변화가 적은 프로젝트에 적합
- 단점 : 요구사항 변경에 대한 유연성 부족, 결과물 확인의 지연, 위험 요소의 후기 발견


### V 모델
- 폭포수 모델의 변형으로, 테스트 단계를 추가 확장하여 테스트 단계가 분석 및 설계와 어떻게 관련되어 있는지를 나타냄
- 폭포수 모델이 산출물 중심이라면 V 모델은 각 개발 단계를 검증하는데 초점을 두기 때문에 오류 줄일수 있다.
