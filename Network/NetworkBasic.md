# 1일차 — 네트워크

---

## 🌐 네트워크 (Network)

> 노드들이 연결되어 자원을 공유하는 구조

### 컴퓨터 네트워크 (Computer Network)
컴퓨터 및 여러 장치들이 연결되어 데이터 및 자원을 공유하고 교환하는 구조

- **데이터** → `0` 전기없음 / `1` 전기있음

---

## 📡 ARPAnet

- 최초의 **패킷 스위칭 네트워크** (Packet Switching)
- 현대 인터넷의 원형

| 방식 | 설명 |
|------|------|
| **회선 교환 방식** (Circuit Switching) | 통신 전용 회선을 점유하는 방식 |
| **패킷 스위칭 방식** (현대 방식) | 패킷 안쪽에 목적지를 적어 놓는 방식. 목적지를 알아서 우회해서 보내는 등 안정성이 높음 |

---

## 🔗 프로토콜 (Protocol)

네트워크에서 데이터가 올바르게 전송·사용되기 위한 **규칙, 약속**

- 오류 없는 통신을 위해 **같은 프로토콜**을 사용해야 한다
- 예: `http ↔ http` 같은 프로토콜끼리 통신
- `0011 0010` 이라도 프로토콜마다 4개씩, 8개씩 해석 방식이 다르다

| 프로토콜 | 역할 |
|----------|------|
| **IP** | 주소 역할 |
| **TCP** | IP의 신뢰성을 책임짐. 상대방이 잘 받았는지 하나하나 확인하는 꼼꼼한 우체부 |

## 🌍 인터넷 (Internet)

전 세계의 컴퓨터 네트워크가 서로 연결된 **거대한 네트워크**

- **TCP/IP**를 기반으로 상호 데이터를 교환한다

---

## 🏗️ 하이어라키클 3계층 (Hierarchical 3 Layer)

복잡한 네트워크 구조를 **3개의 계층**으로 나누어 설명하는 모델

```
┌─────────────────────────────┐
│        Core Layer           │  ← 고속 데이터 처리
├─────────────────────────────┤
│     Distribution Layer      │  ← 보안 정책 / 트래픽 제어
├─────────────────────────────┤
│        Access Layer         │  ← 엔드 디바이스 연결
└─────────────────────────────┘
```

### 1. Access Layer (액세스 계층)
- 엔드 디바이스가 네트워크에 **최초로 연결**되는 계층
- 연결된 여러 엔드 디바이스에 데이터를 올바르게 전송하는 역할 수행

### 2. Distribution Layer (분산 계층)
- 액세스 계층의 장비를 모아 **코어 계층으로 연결**하는 계층
- 보안 정책, 트래픽을 제어하는 작업이 이루어진다

### 3. Core Layer (코어 계층)
- 액세스·분산 계층에서 오는 트래픽을 **최종적으로 처리**하는 장치
- 대량의 데이터를 **빠르고 안정적**으로 처리한다

## 🧩 네트워크 구성 요소

| # | 구성 요소 | 설명 |
|---|-----------|------|
| 1 | **엔드 디바이스** (End Device) | 네트워크의 양 끝 단에 위치한 장치. 사용자가 직접 접근하여 사용하며, 데이터의 송신지·수신지 역할 수행 |
| 2 | **네트워크 장비** | 데이터 송수신, 네트워크 유지/관리를 위해 필요한 하드웨어 장치. 네트워크 인프라의 핵심 역할 |
| 3 | **전송 매체 (Media)** | 데이터가 다른 장치로 이동하는 경로. 유선·무선으로 분류. 성능·안정성·비용을 고려해 선택 |
| 4 | **프로토콜 (Protocol)** | 서로 다른 장치들이 통신하기 위해 지켜야 할 규약. 데이터의 전송·해석 방법을 규정 |

---

## 📶 네트워크의 종류

| 종류 | 이름 | 범위 | 특징 |
|------|------|------|------|
| **LAN** | Local Area Network | 근거리 (사무실, 건물 내부) | 비교적 좁은 범위의 네트워크 |
| **WAN** | Wide Area Network | 광역 (도시·국가·전 세계) | 여러 LAN이 연결된 형태. ISP에 의해 관리 |
| **PAN** | Personal Area Network | 개인 (수 미터 이내) | 개인 장치끼리 연결. 블루투스, 스마트폰 테더링 등 |
| **MAN** | Metropolitan Area Network | 도시 규모 | LAN보다 넓고 WAN보다 좁은 범위 |

## 🔀 토폴로지 (Topology)

네트워크 장치들이 연결되는 **물리적·논리적 구조**  
네트워크를 이루는 요소들을 어떻게 배치하고 연결할 것인지를 정의한다.

### LAN 토폴로지

#### 1. 링형 (Ring)

노드들이 이웃 장치와 연결되어 **원의 형태**로 구성된 방식  
데이터가 목적지에 도착할 때까지 **한 방향으로 순환**하며 전송한다.

| 구분 | 내용 |
|------|------|
| ✅ **장점** | 데이터 충돌이 적다 / 설치와 구성이 간단하다 |
| ❌ **단점** | 네트워크가 클수록 성능이 낮아진다 / 하나의 노드만 문제가 생겨도 전체 네트워크가 영향을 받는다 |


## 🔗 네트워크 토폴로지

### 2. 성형 (Star)

- 네트워크의 모든 노드가 하나의 중앙 장치에 **1:1 연결**되는 방식
- 데이터가 목적지까지 도달하기 위해서는 반드시 중앙 장치를 거쳐야 한다.

| 구분 | 내용 |
|------|------|
| ✅ 장점 | 한 개의 노드에 장애가 발생해도 다른 노드에 영향을 미치지 않는다. |
| ✅ 장점 | 물리적인 구조가 직관적이다. 문제 발생 시 원인을 쉽게 찾을 수 있다. |
| ✅ 장점 | 장치의 추가 및 제거가 간단하다. |
| ❌ 단점 | 중앙 장치에 의존하여 통신하므로 중앙 장치에 장애가 발생하면 전체 네트워크가 다운된다. |

---

### 3. 매시형 (Mesh)

- 네트워크의 모든 노드가 서로 **개별적으로 연결**되는 방식
- 데이터 전송 시 여러 경로를 선택할 수 있어 장애 발생 시 매우 유연하게 대처할 수 있다.

| 구분 | 내용 |
|------|------|
| ✅ 장점 | 네트워크의 신뢰성이 매우 높다. |
| ✅ 장점 | 여러 경로로 트래픽 분산이 가능해 병목 현상을 줄일 수 있다. |
| ❌ 단점 | 모든 노드를 직접 연결해야 하므로 케이블, 장치 수가 많아 구축 비용이 비싸다. |
| ❌ 단점 | 네트워크 구성, 관리가 복잡하다 |

---

### 4. 트리형 (Tree)

- 스타형 구조를 계층적으로 확장한 형태, 최상위 노드를 중심으로 하위 노드들이 트리처럼 연결된 구조

| 구분 | 내용 |
|------|------|
| ✅ 장점 | 네트워크를 계층적으로 관리할 수 있다. |
| ✅ 장점 | 새로운 노드를 추가하는 확장 작업이 용이하다. |
| ✅ 장점 | 특정 하위 그룹의 장애가 전체 네트워크에 영향을 주지 않는다. |
| ❌ 단점 | 상위 계층의 장애에 취약하다 |
| ❌ 단점 | 상위 노드로 트래픽이 몰릴 경우 병목 현상이 발생할 수 있다. |

---

### 5. 버스형 (Bus)

- 하나의 **공통회선(Bus)** 에 모든 노드가 연결되는 방식
- 한 노드가 전송한 데이터는 버스를 통해 모든 노드로 전송된다.

| 구분 | 내용 |
|------|------|
| ✅ 장점 | 구조가 단순하고 필요한 케이블의 양이 적어 설치 비용이 저렴하다 |
| ❌ 단점 | 공통 회선(Bus)에 장애가 발생하면 전체 네트워크가 다운된다 |
| ❌ 단점 | 연결된 노드가 많아질수록 데이터의 충돌 가능성이 높아지게 된다. |
