
##  HDD 관리

### HDD 작업 순서

```
1. 하드 디스크 추가 (SCSI 방식)
2. HDD 이름 확인
3. 파티션 작업 (fdisk)
4. 포맷 (mkfs)
5. 마운트 (mount)
```

### HDD 이름 확인

```bash
fdisk -l <hdd 장치명>
```

### 파티션 작업 (fdisk)

```bash
fdisk /dev/sdb
```

| 명령 | 동작 |
|------|------|
| `d` | 파티션 삭제 |
| `n` | 파티션 추가 |
| `p` | 파티션 테이블 확인 |
| `q` | 저장 없이 종료 |
| `w` | 저장 후 종료 |

### 포맷 (mkfs)

```bash
sudo mkfs.ext4 /dev/sdb1
# 또는
sudo mkfs -t ext4 /dev/sdb1
```

### 마운트 (mount)

```bash
# 마운트 연동
sudo mount /dev/sdb1 /mp/

# 마운트 확인
sudo df -h

# 마운트 해제
sudo umount /dev/sdb1
# 또는
sudo umount /mp/
```

> M.P(Mount Point) : 사용되지 않는 빈 디렉터리를 HDD 장치와 연동하는 디렉터리

### 마운트 유의사항

1. M.P는 **사용되지 않는** 임의의 디렉터리여야 함
2. mount/umount 명령 시 현재 경로가 M.P 디렉터리여서는 안 됨 (사용 중 변경 불가)
3. 하나의 파티션은 반드시 **하나의 M.P** 만 사용 (1:1 대응)
4. 포맷 및 마운트 대상은 **주 파티션(P), 논리 파티션(L)** 만 가능 (확장 파티션(E) 불가)
5. 파티션 작업 시 반드시 마운트 연동을 해제하고 진행


## 1. 파티션 종류

| 번호 | 종류 |
|------|------|
| 1 | 주 파티션 (Primary) |
| 2 | 확장 파티션 (Extended) |
| 3 | 논리 파티션 (Logical) |

## 2. 오토 마운트 (Auto Mount)

> 파일 시스템을 **자동으로 마운트**해주는 기능

- 시스템 종료 과정에서 모든 마운트된 장치는 마운트가 해제됨
- 부팅 과정에서 오토 마운트 처리된 장치만 마운트되어 부팅됨

### 오토 마운트 방식

| 방식 | 설명 |
|------|------|
| 기본 설정 파일 | `/etc/fstab` 을 이용한 오토 마운트 |
| 서비스 이용 | `autofs` 서비스를 이용한 오토 마운트 |

---

### /etc/fstab 구성

```bash
vi /etc/fstab
```

| 필드 | 설명 |
|------|------|
| `<file system>` | 장치명, 오토 마운트할 파일의 이름을 절대경로로 작성 |
| `<mount point>` | 오토 마운트에 사용할 디렉터리(마운트 포인트) 작성 |
| `<type>` | 오토 마운트에 사용할 장치의 파일 시스템 형식 작성 |
| `<options>` | 마운트 옵션 (`defaults` 옵션으로 기본 옵션 전체 적용 가능) |
| `<dump>` | dump 동작 여부 (파일 시스템 백업 도구) |
| `<pass>` | fsck 동작 여부 (파일 시스템 체크 도구) |

> ⚠️ fstab 설정 오류 시 : `exec /sbin/init` (응급 모드 리부트)  
> 안 될 경우 물리적 종료

## 3. NFS (Network File System)

> 서로 다른 시스템 간 파일 및 디렉터리를 공유하기 위한 시스템  
> 네트워크를 통하여 다른 장치의 파티션을 마운트할 수 있다

### 서버 설정

| 항목 | 값 |
|------|----|
| 패키지 이름 | `nfs-kernel-server` |
| 방화벽 포트 | `2049/tcp` |
| 설정 파일 | `/etc/exports` |

### /etc/exports 설정 예시

```
/nfs_server     192.168.108.20(rw,no_root_squash,no_subtree_check,sync)
```

| 항목 | 설명 |
|------|------|
| `/nfs_server` | NFS로 공유할 디렉터리 (디렉터리 단위로 공유) |
| `192.168.108.20` | NFS 파일 시스템을 공유받을 클라이언트 IP |
| `192.168.108.0/24` | 서브넷 마스크 `255.255.255.0` 표기 방식 |

### NFS 옵션 설명

| 옵션 | 설명 |
|------|------|
| `rw` | 읽기 & 쓰기 허용 |
| `ro` | 읽기 전용 |
| `root_squash` | 클라이언트의 root 권한을 인정하지 않음 |
| `no_root_squash` | 클라이언트의 root 권한을 인정 |
| `all_squash` | 클라이언트의 일반 사용자 권한을 인정하지 않음 |
| `no_all_squash` | 클라이언트의 일반 사용자 권한을 인정 (서버·클라이언트 양측에 사용자명, UID, GID가 완전히 일치해야 함) |
| `no_subtree_check` | 공유 디렉터리의 하위 디렉터리 명령 전달 시 하위 디렉터리 검사 수행 안 함 |
| `sync` | 서버·클라이언트 간 데이터 동기화 |

> ⚠️ 권한 인정 X 상태이면 익명 사용자 **Nobody / NoGroup** 으로 취급

---

### 클라이언트 설정

| 항목 | 값 |
|------|----|
| 패키지 이름 | `nfs-common` |

```bash
# NFS 마운트 명령어
mount -t nfs [서버IP]:[공유 디렉터리] [마운트 포인트]

# 예시
mount -t nfs 192.168.108.10:/nfs_server /nfs_client
```
