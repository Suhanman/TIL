# 운영체제의 구성

## 개요

운영체제는 영역별로 **User Space**와 **Kernel Space**로 나뉘어지고, 구성요소로는 인터페이스, 시스템콜, 커널, 드라이버로 구성된다.

---

## 구성 요소

### 1. 인터페이스

OS에서 사용자가 시스템과 상호 작용하고, 컴퓨터 프로그램을 실행하고, 파일을 관리하고, 설정을 구성할 수 있는 CLI/GUI를 제공한다.

### 2. 시스템 콜 (System Call)

응용 프로그램이 운영체제(OS)의 핵심인 커널(Kernel)에게 서비스를 요청하는 **유일한 공식 통로**이다.

**역할:**
- 애플리케이션이 하드웨어와 직접 상호작용하는 것을 막아 시스템의 안정성과 보안 보장
- 파일 입출력, 메모리 관리, 프로세스 관리 등 다양한 작업 수행 인터페이스 제공
- 커널 스페이스에서 실행되며, 호출 시 커널 모드로 변환되어 작업 처리

### 3. 커널 (Kernel)

OS의 핵심 구성 요소로, 중요한 시스템 프로세스를 관리하고 소프트웨어와 하드웨어 간의 상호 작용을 가능하게 한다.

**주요 기능:**
- I/O, CPU 관리, 장치 드라이버, 파일 시스템, 네트워크 통신 처리
- 컴퓨터의 물리적 자원과 추상화 자원 관리

**추상화:**
물리적으로 하나뿐인 하드웨어를 여러 사용자들이 번갈아 사용할 수 있도록 마치 여러 개처럼 보이게 하는 기술. 각 사용자가 하드웨어를 독점하는 것처럼 느끼게 한다.

> 커널은 사용자가 System Call을 통해 컴퓨터 자원을 사용할 수 있게 해주는 자원 관리자이다.

### 4. 드라이버 디바이스 (Driver Device)

커널 스페이스에 포함되지만 특수한 성격을 가진다. 하드웨어를 제어하기 위한 기능을 담고 있는 layer이다.

**드라이버 종류:**
- **문자 드라이버**: 키보드처럼 입출력 단위가 문자인 장치
- **블록 드라이버**: 하드디스크처럼 입출력 단위가 블록인 장치
- **네트워크 드라이버**: 네트워크 패킷을 주고받는 장치 제어

---

## User Space vs Kernel Space

### User Space
프로세스, 애플리케이션, 라이브러리 등 사용자 레벨의 프로그램들이 실행되는 메모리 영역.
- 인터페이스가 유저 스페이스에 해당

### Kernel Space
커널이 실행되는 메모리 영역.
- 커널, 드라이버, 시스템 콜이 커널 스페이스에 해당

---

## Linux 커널 아키텍처

### 1. User Space → System Call Interface
프로세스들이 파일 읽기, 메모리 할당, 네트워크 통신 등의 작업이 필요할 때 시스템콜 인터페이스를 통해 커널에 요청을 보낸다.

### 2. Kernel Space

#### File System
- ext2fs, xiafs, ext4, nfs 등 다양한 파일 형식 지원
- 메모리 관리자와 협력하여 디스크의 파일을 메모리에 로드/언로드
- 블록 디바이스와 직접 연동되어 실제 Disk I/O 수행

#### Memory Manager
- **가상 메모리**: 실제 RAM보다 더 큰 가상 주소 공간 제공. 프로세스에게 독립적이고 연속적인 메모리 공간이라는 환상을 제공
- **페이지 폴트 처리**: 필요한 데이터가 메모리에 없다면 디스크에서 로드
- **Swap**: 메모리가 부족하면 디스크에 임시 저장 (성능 저하)
- File System과 협력하여 캐싱된 데이터 효율적 관리

#### Task Manager
- 여러 프로세스의 CPU 시간 분배 (스케줄링)
- 프로세스의 생성, 삭제, 상태 관리 전담

#### Network Manager
- IPv4, 이더넷, DHCP 등 네트워크 프로토콜 관리
- 디바이스 관리자의 네트워크 디바이스와 통신 담당

#### Device Manager (Driver)
- 각종 장치를 디바이스 드라이버를 통해 일관되게 접근
- **블록 디바이스**: HDD, CDROM, SCSI 등
- **캐릭터 디바이스**: ISDN, Network 등 순차 처리가 필요한 장치
- 실제 하드웨어와의 직접적인 인터페이스

### 3. Kernel Space ↔ Hardware

**Hardware Interface (머신 인터페이스)**는 커널과 하드웨어 사이의 다리 역할

**하향식 (Top-Down)**: 커널의 명령을 하드웨어에 전달
- 디바이스 관리자 → 하드드라이브에 데이터 쓰기
- 네트워크 관리자 → NIC(네트워크 카드)로 패킷 전송

**상향식 (Bottom-Up)**: 하드웨어의 이벤트를 커널에 알림
- **인터럽트**: 디스크 I/O 완료, 네트워크 패킷 도착 등의 신호
- CPU는 현재 작업을 중단하고 인터럽트 핸들러 실행

### 4. Hardware Layer
실제 물리적 장치들의 계층

## 파일 읽기 실행 예제

프로세스가 `read()` 시스템콜을 실행하면:

1. **프로세스**: 파일 읽기 시스템콜 실행 (User Space)
2. **System Call**: 요청을 커널로 전달
3. **File System**: 어느 Disk Block에서 읽을지 결정
4. **Memory Manager**: 해당 데이터가 메모리에 있는지 확인
   - 없다면 디스크 읽기 요청을 디바이스 관리자로 전달
5. **Device Manager**: 디스크 컨트롤러에 명령을 내리고 머신 인터페이스 경유
6. **Hardware**: 디스크에서 데이터를 읽고 인터럽트로 완료 신호 전송
7. **Kernel**: 데이터를 메모리에 로드해서 유저 프로세스로 반환

이렇게 운영체제의 각 영역들이 서로 협력하여 사용자의 요청을 처리한다.
