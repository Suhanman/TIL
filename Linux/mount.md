## 1. 오토 마운트 (Auto Mount)
파일 시스템을 자동으로 마운트해 주는 기능입니다.

### 💡 주요 특징
* 시스템 종료 시 모든 마운트된 장치는 해제됩니다.
* 부팅 과정에서 **오토 마운트 설정된 장치**만 자동으로 다시 마운트됩니다.
* **설정 방법:**
    * 기본 설정 파일 이용: `/etc/fstab`
    * 서비스 이용: `autofs`

### 🛠 `/etc/fstab` 구성 요소
| 항목 | 설명 |
| :--- | :--- |
| **`<file system>`** | 장치명 또는 오토 마운트할 파일의 절대 경로 |
| **`<mount point>`** | 오토 마운트에 사용할 디렉터리 (연결 지점) |
| **`<type>`** | 장치의 파일 시스템 형식 (ext4, xfs, nfs 등) |
| **`<options>`** | 마운트 옵션 (기본값 적용 시 `defaults` 사용) |
| **`<dump>`** | 파일 시스템 백업 도구(dump) 동작 여부 |
| **`<pass>`** | 파일 시스템 체크 도구(fsck) 동작 여부 |

## 2. NFS (Network File System)
서버와 클라이언트 등 서로 다른 시스템 간에 파일 및 디렉터리를 공유하기 위한 시스템입니다.

### 📋 서비스 정보
* **패키지 이름:** `nfs-kernel-server`
* **방화벽 포트:** `2049/tcp`
* **설정 파일:** `/etc/exports`



### ⚙️ 설정 파일(`/etc/exports`) 상세 옵션
NFS는 **디렉터리 단위**로 시스템을 공유합니다.

* **공유 대상:** `/nfs_server` (공유할 디렉터리)
* **클라이언트:** `192.168.108.20` (공유받을 대상 IP)

| 옵션 | 설명 |
| :--- | :--- |
| **rw / ro** | 읽기/쓰기 권한(`rw`) 또는 읽기 전용(`ro`) 설정 |
| **root_squash** | 클라이언트가 접근 시 root 권한 인정 안 함 (익명 취급) |
| **no_root_squash** | 클라이언트가 접근 시 root 권한 인정 함 |
| **all_squash** | 모든 사용자의 권한을 인정하지 않음 (익명 취급) |
| **no_all_squash** | 사용자 권한 인정 (단, 서버/클라이언트 간 사용자명, UID, GID 일치 필요) |
| **no_subtree_check** | 하위 디렉터리에 대한 보안 검사를 수행하지 않음 |
| **sync** | 서버와 클라이언트 간의 데이터를 실시간으로 동기화 |

> **참고:** 권한 인정 안 함(`squash`) 설정 시 해당 사용자는 **Nobody**, **Nogroup**으로 취급됩니다.

## 3. NFS 클라이언트 설정

### 📦 필요 패키지
* `nfs-common`

### 💻 마운트 명령어
클라이언트에서 서버의 공유 디렉터리를 연결할 때 사용합니다.

```bash
# 기본 형식
mount -t nfs [서버IP]:[공유 디렉터리] [마운트 포인트]

# 사용 예시
mount -t nfs 192.168.108.10:/nfs_server /nfs_client
