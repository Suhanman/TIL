# RAID (Redundant Array of Independent Disks)
## 독립 디스크의 이중화 집합/배열

여러 개의 하드 디스크를 하나의 디스크로 결합하여 데이터를 저장하고 관리하는 기술.

**목적:**
- 데이터의 안정성과 성능을 향상시키기 위해 사용
- RAID 구성 방식에 따라서 성능과 용량이 바뀜

---

## RAID Level 구분

| RAID Type | 설명 | 상태 |
|-----------|------|------|
| Linear RAID | 선형 레이드 | 사용 O |
| RAID 0 | Striping | 사용 O |
| RAID 1 | Mirroring | 사용 O |
| RAID 2 | - | 현재 사용 X |
| RAID 3 | 패리티 비트 | 현재 사용 X |
| RAID 4 | 패리티 블럭 | 현재 사용 X |
| RAID 5 | Striping With Parity | 사용 O |
| RAID 6 | Striping With Dual Parity | 사용 O |

---

## 1) Linear RAID (선형 RAID)

**특징:**
- 최소 2개 이상의 디스크 필요
- 데이터가 순차적으로 여러 디스크에 분산되어 저장 (선형으로 저장)
- 여러 개의 디스크가 하나의 볼륨으로 결합되어 사용

**위험성:**
- 한 개의 디스크만 고장나도 모든 디스크를 사용할 수 없음

**저장 구조:**
```
DISK1  |  DISK2
-------|-------
  1    |   6
  2    |   7
  3    |   8
  4    |  ...
  5    |
```

---

## 2) RAID 0

**특징:**
- 최소 2개 이상의 디스크 필요
- 데이터를 블록으로 쪼개어 각각 다른 디스크에 나누어 저장
- 데이터가 병렬로 저장되어 **성능이 뛰어남**

**위험성:**
- 한 개의 디스크만 고장나도 모든 디스크를 사용할 수 없음

**저장 구조:**
```
DISK1  |  DISK2
-------|-------
  1    |   2
  3    |   4
  5    |   6
  7    |   8
 ..    |  ..
```

---

## 3) RAID 1

**특징:**
- 최소 2개의 디스크 사용
- 모든 데이터가 중복되어 저장

**용량:**
- 디스크 전체 용량의 절반밖에 사용하지 못함
- 예) DISK1 (10G) + DISK2 (10G) = 10G

**안정성:**
- 디스크 하나가 고장나도 중복되어 저장된 디스크가 하나 더 있어 **매우 안전함**

**저장 구조:**
```
DISK1  |  DISK2
-------|-------
  1    |   1
  2    |   2
  3    |   3
  4    |   4
 ...   |  ...
```

---

## 4) RAID 4

**특징:**
- 최소 3개의 디스크 사용
- 데이터를 블록으로 쪼개어 각각 다른 디스크에 저장
- 디스크에 오류가 발생하면 패리티를 사용하여 복구 가능
- 디스크 하나를 패리티 디스크로 사용

**단점:**
- 패리티 디스크의 수명이 짧아져 RAID 5 방식이 더 선호됨

**저장 구조:**
```
DISK1  |  DISK2  |  DISK3(Parity)
-------|---------|----------------
  1    |   2     |      P
  3    |   4     |      P
  5    |   6     |      P
  7    |   8     |      P
```

**패리티 계산 예:**
```
DISK1  +  DISK2  =  DISK3(Parity)
  1    +    2    =        3
  3    +    4    =        7
  5    +    6    =       11
  7    +    8    =       15
```

**복구 예:**
```
DISK1  +  DISK2  =  DISK3(Parity)
  1    +    ?    =        3
  3    +    ?    =        7
  5    +    ?    =       11
  7    +    ?    =       15
```

---

## 5) RAID 5

**특징:**
- 최소 3개의 디스크 사용
- 데이터를 블록으로 쪼개어 각각 다른 디스크에 저장
- 디스크에 오류가 발생하면 패리티를 사용하여 복구 가능

**개선 사항:**
- RAID 4의 단점을 개선한 방식
- 매번 다른 디스크에 패리티를 저장 (부하 분산)

**저장 구조:**
```
DISK1  |  DISK2  |  DISK3
-------|---------|-------
  1    |   2     |   P
  3    |   P     |   4
  P    |   5     |   6
  7    |   8     |   P
 ...   |  ...    |  ...
```

### 짝수/홀수 패리티 (Even/Odd Parity)

| DISK1 | DISK2 | DISK3 | 짝수 P | 홀수 P |
|-------|-------|-------|--------|--------|
|   1   |   0   |   P   |   1    |   0    |
|   1   |   1   |   P   |   0    |   1    |
|   0   |   0   |   P   |   0    |   1    |
|   0   |   1   |   P   |   1    |   0    |

- **짝수 패리티:** 데이터 비트들의 합이 짝수가 되도록 하는 방법
- **홀수 패리티:** 데이터 비트들의 합이 홀수가 되도록 하는 방법

---

## 6) RAID 6

**특징:**
- 최소 4개의 디스크 사용
- 패리티를 두 개 사용

**안정성:**
- 디스크가 2개 고장나도 복구 가능
- 안정성이 더 올라감

**트레이드오프:**
- 공간 효율은 저하됨
- 패리티를 두 개 생성하는 만큼 알고리즘이 복잡해 쓰기 성능은 떨어짐

## RAID 레벨별 최소 디스크 수

| RAID Level | 최소 디스크 수 |
|-----------|--------------|
| Linear | 2 |
| RAID 0 | 2 |
| RAID 1 | 2 |
| RAID 5 | 3 |
| RAID 6 | 4 |

## mdadm 명령어

Linux에서 RAID를 관리하기 위한 명령어입니다.

**사용 예:**
```bash
mdadm --create /dev/md10 --level=linear --raid-devices=2 /dev/sdb1 /dev/sdc1
```

**명령어 옵션:**
- `--create`: RAID 배열 생성
- `--level`: RAID 레벨 지정 (linear, 0, 1, 5, 6 등)
- `--raid-devices`: 사용할 디스크 개수
